## 1、正常大屏的响应式布局方案
vw 和 addEventListener+scale
可能会形成scale+响应式布局的方案

大屏其实不适合vw方案，更多的使用scale
但是一个项目中可能包含多端，比如无人餐厅大屏+管理平台
象棋大屏+h5

scale方案的缺点
不适合h5
不是真正的响应式
会影响：fixed sticky 鼠标坐标 canvas tooltip
缩放的时候可能模糊
出现黑边

scale+局部响应式方案：
| 区域   | 方案    |
| ---- | ----- |
| 整体画布 | scale |
| 卡片内部 | flex  |
| 图表   | 自适应   |
| 字体   | clamp |
| 超宽屏  | 居中留黑  |

## 2、多设备在一个项目里如何布局
目前象棋大屏、h5 pad页面都在一个项目里
主体是vw布局 展示象棋大屏
h5基本是按照375px宽度开发的 后续优化可以加上scale但是考虑对canvas是否有影响
pad采用flex布局 其实和大屏差不多
